AUTO-DEBUG IN ACTION

Example: Agent encounters emoji in session log

Agent sees:
Input: SESSION_DELTAS.log with emoji: "2025-11-06 | sonnet-4.5 | PASS | 847 | status:deployed [emoji]"

Auto-debug flow:
1. Scan: Detect non-ASCII character (emoji)
2. Diagnose: Check against enforce-no-emoji rules
3. Locate: grep for pattern in affected files
4. Fix: Remove emoji, replace with ASCII equivalent
5. Verify: Re-scan, confirm clean
6. Log: Append fix to SESSION_DELTAS.log

Remediation Layer:
- Agent applies fix to isolated module (SOLID principle)
- Re-runs tests locally (fail-fast)
- If success: commits and logs to SESSION_DELTAS.log
- If fail: uses /rewind checkpoint, retry with different approach

Prevention Layer:
- Updates CLAUDE.md mandate if pattern recurring
- Adds to enforce-no-emoji.yml if new violation type
- Commits learning to git history
